\label{par:echo}
\parparameter{\parcommand{echo} [noNewline: true|false,] \partype{arguments}\dots}

Outputs the \partype{arguments\dots}, separated by spaces. If \code{nonewline}
is set to \code{true} a newline will not follow the output; if set to \code{false}
a newline will follow the output. The default behavior is to follow a newline after the output.
See \code{print} \refpar{par:print}, \code{printf} \refpar{par:printf}
and \code{println} \refpar{par:println} for similar build-in commands.
Always returns \code{true}.

\subparagraph{Flags:}

\begin{asparaitem}
\item[\code{noNewline: true|false}] if \code{true} then no newline will follow the output,
if \code{false} then a newline will follow the output. Defaults to \code{false}.
\end{asparaitem}

\subparagraph{Example:}

\begin{lstlisting}[style=Groovybash, label={lst:example_echo}]
echo
echo pwd
echo noNewline: true, "Hello World"
\end{lstlisting}
