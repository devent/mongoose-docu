\label{par:parse}
\parparameter{\parcommand{parse} \partype{obj} [, \partype{args}]}

Parse the specified command line arguments \partype{args}. If no arguments
are specified it will use the command line arguments of the script, stored in
the \parcommand{ARGS}, see \refpar{par:args}, build-in variable.
\partype{obj} needs to be an instance of a Java bean class annotated
with \code{Option} or \code{Argument}.

\subparagraph{Return Value:}

The return value is an instance of \code{ParsedReturnValue}. It offers
seamless access to the properties of the used Java bean class. It also
offers methods to generate an example or to generate an usage of the parameter.

\subparagraph{Example:}

See the \reflst{lst:script_parse_cl_parameter} for an example how to define
the parameter bean class. The \reflst{lst:script_print_usage_cl_parameter}
shows how a generated example and generated usage can be printed to the
standard output.

\begin{lstlisting}[style=Groovybash, label={lst:example_args_parse}]
parser = parse new Parameter()
parser = parse new Parameter(), ARGS
\end{lstlisting}
